Subscribe to the position stream to see changes to your position data in **real-time**.

**Topic:** `position`

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|id |string |Message ID|
|topic |string |Topic name |
|creationTime |number |Data created timestamp (ms) |
|data |array |Object |
|> [category](../../enum#category) |string |Product type|
|> symbol |string |Symbol name |
|> side |string |Position side. `Buy`,`Sell` |
|> size |string |Position size |
|> [positionIdx](../../enum#positionidx) |integer |Used to identify positions in different position modes|
|> tradeMode | integer |Trade mode <ul><li>USDT perpetual & Inverse contract: `0`: cross-margin, `1`: isolated margin</li><li>Trading account deprecated, always `0`. Please query account config to get actual margin mode</li></ul>|
|> positionValue |string |Position value |
|> riskId |integer |Risk limit ID.|
|> riskLimitValue |string |Risk limit value corresponding to riskId.|
|> entryPrice |string |Entry price |
|> markPrice |string |Mark price |
|> leverage |string |Leverage.|
|> positionBalance |string |Position margin.|
|> autoAddMargin | integer |Whether to add margin automatically. `0`: false, `1`: true.|
|> positionMM |string |Position maintenance margin. <ul><li>Classic account: ignore this field</li><li>Unified account Portfolio Margin (PM): returns empty string</li></ul>|
|> positionIM |string |Position initial margin. <ul><li>Classic account: ignore this field</li><li>Unified account Portfolio Margin (PM): returns empty string</li></ul>|
|> liqPrice | string  | Position liquidation price. <ul><li>Normal account & Unified account (isolated): it is the real price for isolated and cross positions; when `liqPrice <= minPrice` or `liqPrice >= maxPrice`, it returns ""</li><li>Unified account (cross): it is an estimated price for cross positions (risk rate is controlled at account level); when `liqPrice <= minPrice` or `liqPrice >= maxPrice`, it returns ""</li><li>For Portfolio Margin mode, this field is empty and no liquidation price is provided</li></ul> |
|> bustPrice |string |Est.bankruptcy price. |
|> tpslMode |string |Depreciated, meaningless here, always "Full". |
|> takeProfit |string |Take profit price |
|> stopLoss |string |Stop loss price |
|> trailingStop |string |Trailing stop |
|> unrealisedPnl |string |Unrealised profit and loss |
|> cumRealisedPnl |string |Cumulative realised PnL |
|> isReduceOnly | boolean | Useful when the Zoomex lowers a symbol's risk limit. <ul><li>`true`: only reduce position is allowed. Consider lowering risk tier, decreasing leverage or position size, adding margin, or cancelling orders. After these operations, you can call "Confirm New Risk Limit"</li><li>`false` (default): no restriction; your position is under the system risk level</li><li>Only meaningful for futures isolated & cross positions</li></ul> |
|> mmrSysUpdatedTime | string | Useful when the Zoomex lowers a symbol's risk limit. <ul><li>When isReduceOnly=`true`: timestamp when the system forcibly adjusted MMR</li><li>When isReduceOnly=`false`: if not empty, the timestamp when MMR adjustment has been completed</li><li>Only set for system adjustments; manual adjustments will not show timestamp here</li><li>Defaults to ""; if this symbol had system downgrade before, it shows the last system operation time</li><li>Only meaningful for futures isolated & cross positions</li></ul> |
|> leverageSysUpdatedTime | string | Useful when the Zoomex lowers a symbol's risk limit. <ul><li>When isReduceOnly=`true`: timestamp when the system forcibly adjusted leverage</li><li>When isReduceOnly=`false`: if not empty, the timestamp when leverage adjustment has been completed</li><li>Only set for system adjustments; manual adjustments will not show timestamp here</li><li>Defaults to ""; if this symbol had system downgrade before, it shows the last system operation time</li><li>Only meaningful for futures isolated & cross positions</li></ul> |
|> [positionStatus](../../enum#positionstatus) |string |Position status. `Normal`, `Liq`, `Adl` |
|> [adlRankIndicator](../../enum#adlrankindicator) |integer |Auto-deleverage rank indicator. <a href="https://www.zoomex.com/en-US/help-center/s/article/What-is-Auto-Deleveraging-ADL">What is Auto-Deleveraging?</a>|
|> createdTime |string |Position created timestamp (ms) |
|> updatedTime |string |Position data updated timestamp (ms) |

### Subscribe Example
```json
{
    "op": "subscribe",
    "args": [
        "position"
    ]
}
```

```javascript
var url = require('url');
var WebSocket = require('ws');
var crypto = require('crypto');
var endpoint = "wss://stream-testnet.zoomex.com/v3/private"
console.log('attempting to connect to WebSocket %j', endpoint);
var client = new WebSocket(endpoint);
const apiKey="XXXXXXXXX";
const apiSecret="XXXXXXXXX";
client.on('open', function () {
	console.log('"open" event!');
	console.log('WebSocket Client Connected');
	const expires = new Date().getTime() + 10000;
	const signature = crypto.createHmac("sha256", apiSecret).update("GET/realtime" + expires).digest("hex");
	const payload={
		op: "auth",
		args: [apiKey, expires.toFixed(0), signature],
	}
	client.send(JSON.stringify(payload));
	setInterval(()=>{client.ping()}, 30000);
	client.ping();
	client.send(JSON.stringify({"op": "subscribe", "args": ['position']}));
});

client.on('message', function (data) {
	console.log('"message" event! %j', JSON.parse(Buffer.from(data).toString()));
});
client.on('ping', function (data, flags) {
	console.log("ping received");
});
client.on('pong', function (data, flags) {
	console.log("pong received");
});
```
