Query real-time position data, such as position size, cumulative realizedPNL.

:::info

Trading account 2.0 - Query inverse positions

1. You can get all positions with "/v3/position/list?category=inverse".
2. Querying multiple symbols at once is not supported for inverse contracts.

:::

### HTTP Request
GET `/cloud/trade/v3/position/list`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../v3/enum#category) |**true** |string |Product type <ul><li>Unified account: `linear`, `inverse`</li><li>Normal account: `linear`, `inverse`. _Please note that `category` is not involved with business logic and is only used for routing_</li></ul> |
|symbol |false |string |Symbol name<ul><li>If `symbol` passed, it returns data regardless of having position or not.</li><li>If `symbol`=null and `settleCoin` specified, it returns position size greater than zero.</li></ul>|
|baseCoin |false |string |Base coin. |
|settleCoin |false |string |Settle coin. For `linear` & `inverse`, either `symbol` or `settleCoin` is **required**. `symbol` has a higher priority |
|limit |false |integer |Limit for data size per page. [`1`, `200`]. Default: `20`|
|cursor |false |string |Cursor. Use the `nextPageCursor` token from the response to retrieve the next page of the result set|

### Response Parameters

|list|array |Object |

|> avgPrice |string  |Average entry price |

|> tradeMode | integer |Trade mode <ul><li>Classic account: `0` cross margin, `1` isolated margin</li><li>Unified account (except inverse): deprecated, always `0`. Please query account config to get the actual margin mode</li></ul>|
|> [positionStatus](../v3/enum#positionstatus) | String |Position status. `Normal`, `Liq`, `Adl` |
|> leverage | string  |Position leverage. Valid for contracts only.|
|> markPrice | string  |Last mark price |

It means your position is below the system risk level during adjustment</li><li>Only meaningful for futures isolated and cross positions</li></ul> |
|> mmrSysUpdatedTime | string | Useful when Zoomex lowers the risk limit. <ul><li>When isReduceOnly=<code>true</code>: the timestamp when MMR will be forcibly adjusted by the system</li><li>When isReduceOnly=<code>false</code>: if not empty, the timestamp when the MMR adjustment has been completed by the system</li><li>Only set for system adjustments. Manual adjustments will not show a timestamp here</li><li>Defaults to <code>""</code>. If this symbol has been downgraded by the system previously, it shows the most recent system operation timestamp</li><li>Only meaningful for futures isolated and cross positions</li></ul> |
|> leverageSysUpdatedTime | string | Useful when Zoomex lowers the risk limit. <ul><li>When isReduceOnly=<code>true</code>: the timestamp when leverage will be forcibly adjusted by the system</li><li>When isReduceOnly=<code>false</code>: if not empty, the timestamp when the leverage adjustment has been completed by the system</li><li>Only set for system adjustments. Manual adjustments will not show a timestamp here</li><li>Defaults to <code>""</code>. If this symbol has been downgraded by the system previously, it shows the most recent system operation timestamp</li><li>Only meaningful for futures isolated and cross positions</li></ul> |
|> [adlRankIndicator](../v3/enum#adlrankindicator) |integer |Auto-deleverage rank indicator. <a href="https://www.zoomex.com/en-US/help-center/s/article/What-is-Auto-Deleveraging-ADL">What is Auto-Deleveraging?</a>|

|nextPageCursor | string  |Refer to the `cursor` request parameter |
